# 🌍 Parabola：技術手冊 (v0.5)

這是一個 IPA 導向的語言建構工具，旨在提供從音標選取、頻率管理、單字生成到構詞規則維護的一站式解決方案。

---

## 🏠 首頁：專案管理
* **專案建立**：開始前必須先建立一個專案。
* **更名注意**：目前介面暫不支援專案更名，但可以直接手動更改專案資料夾名稱，系統會自動同步。


## ⚙️ 語法配置指引 (Grammar Configuration Guide)

### 1. 配置邏輯：由宏觀到微觀
本系統採用「層級連動」邏輯，建議依照分頁順序（從上到下）進行設定，以確保自動化腳本能正確觸發數據灌注。

#### A. 形態類型與基本語序 (Morphology & Word Order)
*   **語言架構**：首先定義語言的基礎屬性（如：綜合語或分析語）。
*   **預設語序**：設定 SVO/SOV 等基本語序，這將直接影響後續「構詞管理」中句子生成的預設排列。

#### B. 通用特性 (Universal Features) —— 【核心定義池】
*   **全局定義**：此處是語言特徵的「母體」。若您的語言包含格位、人稱、數、性等變化，**必須在此先行勾選**。
*   **自動灌注 (Feature Injection)**：在此處選定的所有具體值（如：單數、複數、一波人稱），將自動同步至「名詞」、「動詞」、「形容詞」等專屬語法分頁。

#### C. 詞類語法設定 (Noun/Verb/Adj Syntax)
*   **繼承與擴充**：各詞類分頁會自動繼承通用特性的設定。您只需針對該詞類進行微調（如：設定動詞專屬的時態或語態）。
*   **自定義標籤**：若預設選項不足，可利用「+」按鈕新增自定義標籤。這些標籤將同步進入「排序管理」系統。

---

### 2. 排序管理：定義數據權重
此步驟決定了特徵在系統各處（如字典表格、導出文件、構詞表格）的呈現順序。

*   **動態生成**：系統會根據您在配置分頁中「勾選」或「輸入」的所有項目，即時生成可拖拽的排序晶片。
*   **權重影響**：
    *   **表格分佈**：影響屈折變化表（Inflection Table）的橫縱軸排列先後。
    *   **字典排序**：決定字典中語法標籤的顯示順序（例如：讓「格位」優先於「數」顯示）。
*   **操作方式**：直接拖拽晶片即可完成排序。若項目未出現，請檢查是否已在配置分頁中勾選該特徵。

    *   可以。只要標籤內有輸入文字並儲存，它就會自動出現在排序分頁的對應卡片中。
*   **為什麼配列的格位不對？**

## 📊 音標管理 (IPA Management)

### **1. 總覽 (Overview)**
* **字母表順序**：可自定義排序（目前僅影響顯示順序）。
* **出現頻率**：調整各個 IPA 在所屬類別（子音/母音）中的生成頻率。
* **機率算法**：若未指定音節類型，單個音素出現機率為：
  $$P = \frac{\text{IPA.frequency}}{\sum \text{Category.frequency}}$$

### **2. IPA 分類區 (Categories)**
#### 📖 使用說明
* **自動生成**：在「音標系統」儲存後，系統會自動根據發音特徵產生預設分類（如 P, N, F 等）。
* **手動新增**：點擊「新增類別」可建立自定義代碼。
* **編輯模式**：點擊分類塊（**藍色邊框**）即進入編輯狀態。此時點擊下方的 **IPA 符號池** 即可新增或移除該分類中的符號。

#### 📝 欄位定義與規範
* **CODE (代碼)**：用於規則寫作的簡寫（如 `P -> B / V_V`）。建議使用**大寫英文字母**。
* **COMMENT (註解)**：說明代碼意義（如 `Plosives` 或 `脣音`），不參與程式運算。
* **捲動區域**：分類列表固定高度為 **300px**，可直接於內部捲動。
* **特殊限制**：
  * **C、V** 預設為子音與母音，**無法更改**。
  * **代碼唯一性**：CODE 不可重複，否則會彈出警告。
  * **儲存流程**：目前不支援在建立分類時同步調整註解。請**先更新分類並儲存設定，再回來調整註解內容**。

#### 📋 預設分類表
| 代碼 (CODE) | 註解 (COMMENT) | 包含範圍 (發音部位/方法) |
| :--- | :--- | :--- |
| **P** | Plosives | 塞音 (如 p, t, k, b, d, g) |
| **N** | Nasals | 鼻音 (如 m, n, ŋ) |
| **F** | Fricatives | 擦音 (如 f, s, x, v, z) |
| **L** | Liquids/Approximants | 流音、近音、顫音 (如 l, r, j, w) |
| **B** | Labials | 雙唇音、唇齒音 (唇部發音) |
| **K** | Coronals | 齒音、齒齦音、齒齦後音 (舌尖/舌葉發音) |
| **G** | Gutturals/Back | 軟顎音、小舌音、喉音 (後部發音) |
| **S** | Sibilants | 噝音 (具有高頻特徵的擦音) |
| **O** | Others | 其他未歸類之音素 |

## ✍️ 單字生成 (Generator)
系統參照音標管理的權重邏輯，依據音節格式輸入（如 `CVC`）與音節數範圍隨機生成。
1. **批量生成**：生成大量無意義字串，使用者再自行篩選加入字典。
2. **一對一新建**：對應翻譯清單直接生成。目前預設詞性皆為「名詞」，需手動修改。

## 🛠️ 構詞管理 (Morphology Management)

### 1. 維度配置與矩陣生成
本模組的核心功能是將抽象的語法特徵轉化為直觀的**屈折變化表 (Inflection Table)**。系統會根據您的選取自動計算維度組合：

*   **維度來源**：
    *   **Universal (U)**：繼承自「通用特性」的全域特徵（如：格位、人稱）。
    *   **Local**：該詞類專屬的特徵（如：名詞的定指性、動詞的時態）。
*   **動態佈局邏輯**：
    *   **單維度**：生成簡單的垂直清單，適用於僅有單一變化的特徵。
    *   **雙維度 (XY 軸)**：自動將前兩個勾選的維度設為橫縱軸（例如：縱軸為「格」，橫軸為「數」）。
    *   **多維度 (3+) 笛卡兒積**：當勾選三個以上維度時，系統會將第三個起的維度進行全組合運算，並拆分為多個獨立的子矩陣。

### 2. 變化規則錄入 (Marker Input)
*   **詞綴與詞幹變化**：在矩陣空格中輸入對應的語法標記（如：詞尾 `-s`, `-en` 或詞綴 `-ing`）。
*   **快捷導航**：支援 `Enter` 鍵導航，方便您在複雜的表格中快速移動焦點。
*   **數據封裝**：此處填寫的資料將直接與「詞典管理」關聯，作為自動化生成單字屈折形式的基礎規則。

---

### 💡 操作指南與常見問題

*   **為什麼左側配置區是空的？**
    *   系統偵測到您尚未在「語法配置」頁面開啟任何功能。請先前往該頁面勾選所需的語法維度並點擊儲存。
*   **如何切換橫縱軸位置？**
    *   矩陣生成順序取決於勾選順序。若想更換 XY 軸，請先取消勾選，再依照「先 Y 軸、後 X 軸」的順序重新勾選。
*   **關於動詞邏輯**：
    *   **目前狀態**：名詞與形容詞矩陣已完全開放；動詞部分的複雜連動（如：一致性協定與時體氣組合）目前仍在內部測試中，暫時建議以自定義標籤方式處理。
